2. Stack & Design Choices

I chose vanilla HTML, CSS, and JavaScript because this app has no complex 
state or reusable components — adding React or Vue would introduce unnecessary 
complexity with no real benefit. Zero dependencies also means anyone can open 
it in a browser without installing anything.

->Two specific decisions:

1- Timer size: I made the timer display 8rem — large enough to read from 
across the room without squinting. The timer is the only thing that matters 
on this screen, so it dominates the layout intentionally.

2- Color switching: When the mode switches from Focus to Break, the entire 
color scheme changes from red to teal. This means the user knows their current 
mode instantly from color alone — without needing to read the mode label.

3. Responsive & Accessibility

On a 360px phone the layout stacks vertically and stays centered — nothing 
overflows. On a 1440px laptop, max-width 500px with flexbox centering prevents 
the content from stretching too wide and becoming unreadable.

-> Handled: Buttons are native HTML button elements — fully keyboard 
accessible, focusable with Tab, and activatable with Enter or Space. Input 
fields have visible labels attached directly to them.

-> Skipped: I did not implement ARIA live regions, so a screen reader will 
not announce the countdown ticking. I also skipped focus trap management — 
keyboard users can tab outside the app into browser controls. With more time 
I would add aria-live="polite" on the timer display so screen reader users 
get periodic time updates.

4. AI Usage

I used Claude throughout this project in two ways:

First, Claude explained JavaScript concepts to me line by line — setInterval, 
localStorage with JSON.stringify/parse, padStart, the ternary operator, and 
the Web Audio API. I wrote every line of code myself after understanding 
the concept.

Second, I caught and fixed bugs in Claude's suggested code myself. For example, 
Claude generated this line in the saveSession function:

  const duration = isFocusMode ? focusDuration : focusDuration

I noticed both sides of the ternary were identical — making it useless. 
I flagged it and we removed the line entirely. Claude also initially wrote 
padStart("2, 0") as a single string argument — I caught that it should be 
two separate arguments padStart(2, '0') and fixed it.

5. Honest Gap

Two things are not polished enough:

-> Input validation: There is no validation preventing a user from setting 
both minutes and seconds to 0. This would create a 0-second timer that 
instantly completes and loops infinitely. With another day I would add a 
guard in updateDurations() that enforces a minimum total duration of 
at least 10 seconds.

-> Audio: The completion sound is a basic sine wave beep generated by the 
Web Audio API. It works but it's not satisfying. With more time I would 
design a proper multi-tone completion sound — something that feels rewarding 
when a focus session ends.
